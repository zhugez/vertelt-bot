# Roadmap

## Near term (v0.2.x)

- Increase unit/integration coverage for Telegram adapter edge cases
- Validate and document all env var constraints
- Add structured logging fields for better observability

## Mid term (v0.3.x)

- Add webhook mode alongside polling
- Add persistence-backed rate limiting option
- Provide packaged deployment examples for systemd and containers

## Long term (v1.0)

- Plugin-style command extension interface
- Multiple chat platform adapters (Telegram first-class, others optional)
- Hardened production profile with benchmarks and SLO guidance
